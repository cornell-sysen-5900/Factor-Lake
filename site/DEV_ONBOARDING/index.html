    Factor Lake New Member Onboarding Guide

    This guide walks a new contributor through access, setup, and day-to-day workflow in the order they should do it.

    1. Wait for access

    Before you start, the professor must add you to the required workspaces:

    1. Factor-Lake GitHub Repo
    2. Factor Lake Supabase Project
    3. Quant Finance Project Trello
    4. Slack Workspace

    Do not start work until all four are available.

    2. Confirm you can open every tool

    1. Sign in to the GitHub Repo.
    2. Open the Supabase Project.
    3. Open the Trello board.
    4. Open Slack.
    5. Confirm you can read the team channels and project boards.

    3. Read the setup docs in order

    Use these docs as your first local setup path:

    1. Read Documentation Contributor Guide.
    2. Read Contributing.
    3. Read Supabase Setup.
    4. Read Deployment if you need to publish or verify the app.
    5. Read Factor Lake User Guide to understand the app workflow.

    4. Set up your local repository

    1. Clone the Factor-Lake GitHub Repo.
    2. Create a branch before making changes.
    3. Install the project dependencies.
    4. Run the app or tests only after the repository is set up.

    If you need the exact branch-and-commit workflow, follow Contributing.

    5. Understand the main tools

    Tool What it is for What to do first
    GitHub Repo Source code and docs Clone it, create a branch, and read the docs before editing code
    Factor Lake Streamlit App Live portfolio analysis app Open it, load data, and verify the analysis flow
    Supabase Project Backend data source Use it only when data or schema changes are needed
    Trello board Planning and task tracking Check the Reference list and move cards as work changes
    Slack Workspace Team communication Ask questions there instead of guessing
    ScrumPoker Estimation tool Use it during sprint planning to estimate effort
    Streamlit Cloud App hosting Use it to deploy and verify the Streamlit app

    6. Learn the project workflow

    1. Check the Trello board for the story you will work on.
    2. Clarify the story before starting implementation.
    3. Estimate the story with ScrumPoker during the team meeting.
    4. Move the card to Doing when you start.
    5. Open a pull request when your branch is ready.
    6. Move the card to Awaiting PR Approval while it is under review.
    7. After merge, the card becomes ready for demo in the professor meeting.

    7. Use Slack correctly

    1. Use the semester Slack Workspace for questions and coordination.
    2. Post blockers early.
    3. Ask for clarification before editing code or data.
    4. Keep discussions in the semester channel so the whole team stays aligned.

    8. Know where each deeper guide lives

    1. Documentation Contributor Guide explains how to add or edit docs.
    2. Contributing explains the branch-and-commit workflow.
    3. Deployment explains how to publish the app.
    4. Supabase Setup explains how to configure the database connection.
    5. Streamlit Admin Guide explains production app operations.

    9. Your first-week checklist

    1. Confirm you can access all required tools.
    2. Read the setup docs in Section 3.
    3. Clone the repo and open the app locally.
    4. Check the Trello Reference list for project links.
    5. Ask one question in Slack if anything is unclear.

    10. Quick reference

    Need Go here
    Clone and branch correctly Contributing
    Add or edit docs Documentation Contributor Guide
    Configure Supabase Supabase Setup
    Deploy or verify the app Deployment
    Understand the app workflow Factor Lake User Guide
    Ask questions Slack Workspace

    11. Final rule

    If something is already documented, follow the doc instead of improvising. If something is not clear, ask in Slack and update the relevant guide later if the missing step should be documented.
